1 Introduction

In this document I present the correction template for automatic correction of assignments.

The rationale of this autocorrection is that students are given a .Rmd that includes their assignment. The fundamental aspects of this document are:

  1. at the top there is a call to set.seed and the students replace a place holder by their student number. Something like
# replace 11111 with your student number
set.seed(11111)
  1. data are generated such that each student gets their own data, say, mydata, as
n <- round(runif(1,100,200))
mydata <- rnorm(n)
  1. Questions are asked and students answer them by replacing lines of code like the one below
# what is your sample size
myn<-NA
# what is the mean of your data
mymean<-NA

with their results

# what is your sample size
myn<-n
# what is the mean of your data
mymean<-mean(mydata)

  1. At the end of their report a text file is created, with their results and their student number

  2. Students submit their assignments, the .Rmd, the .html (or .pf or .docx) and the .txt

Then, all the assignments are placed in a folder and the correction code is run. The correction code itself loops over students and:

  1. reads the students’ results
  2. recreates all the steps required to produce the correct answers given the student seed
  3. checks students results against the correct answers
  4. merges the results with a list of all students in the course
  5. exports the results in a format that moodle can read so that grades can be released to students

I would like to automatize this as much as possible, maybe even do an R package out of it, to allow others to do the same for their classes/assessments. The problem is that the methods is cumbersome in the sense that one always needs to manually tweak two files:

  1. The .Rmd template, as data will have to be created differently for each new assignment
  2. The auto-correction file (essentially, a file like this one, minus this introduction!), which includes the data creation and the code that creates the correct answers (via function getmyres)

While the data creation could be done via a function that creates data given a single input parameter, i.e. the student number, which could then be called by both the .Rmd template and this file to create the data, obtaining the correct answers will always be fiddly and dependent of the case in hand.

It could however maybe just mean one would have to create a function that again given 1 argument, again the student number, would return the correct results.

Below is an example of the autocorrection code. The file that would be given to the students would be AssignmentTemplate.Rmd.

6 Get students’ results and compare to correct results

# Get student results and compare to true results ---- 

#object to hold whether results are correct - by question
correctres <- matrix(NA,nrow=nfiles,ncol=1+nq)
#objects to hold the grades - by question
grades <- matrix(NA,nrow=nfiles,ncol=1+nq)

#for each student that handed in a .txt
for (i in 1:nfiles){
  # get student results
  studentres <- dget(file=paste0(pathfiles,"/",txtfiles[i]))
  # record student number
  correctres[i,1] <- studentres[[1]]
  # some tweaking required as some students reported the value of a factor
  # for result3 while my correction is a character
if (!is.null(studentres[[4]])) {
studentres[[4]] <- ifelse(is.factor(studentres[[4]]),as.character(studentres[[4]]),studentres[[4]])
}
  # get true results for student i
  # this is the hard stuff, needs to be tweaked each time
  # by creating a bespoke code inside function "getmyres"
  trueres <- getmyres(aluno=studentres[[1]],nq=nq)
  # check correct answers
  # the first element of the comparison corresponds 
  # to the student number, so it is ignored here
  correctres[i,2:(1+nq)] <- purrr::map2_lgl(studentres, trueres, ~ isTRUE(all.equal(.x, .y, tolerance = 1e-7)))[-1]
  # the grade by question, using the weights
  grades[i,] <- weights*correctres[i,]
}
